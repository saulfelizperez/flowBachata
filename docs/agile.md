## ¿Que es Agile y cual es su objetivo?

```Bash
Agile es una forma de desarrollar software basada en la adaptabilidad, la colaboración y la entrega progresiva de resultados.
```

### Objetivo de Agile

```Bash
 Crear software funcional de manera rápida y poder adaptarse a los cambios durante el desarrollo.
```

### Ideas clave

```Bash
- Entregas frecuentes de partes funcionales
- Capacidad de adaptación a cambios
- Comunicación constante con el cliente
- Equipos organizados de forma autónoma
```

---

## 🔹 ¿Qué es Scrum?

```Bash
Scrum es un marco de trabajo dentro de Agile que divide el proyecto en ciclos cortos llamados sprints, permitiendo avanzar de forma iterativa.
```

### Roles en Scrum

```Bash
-- Product Owner

  Define los requisitos del producto y gestiona la lista de tareas.

-- Scrum Master

  Supervisa el proceso y elimina impedimentos.

-- Equipo de desarrollo

  Construye el producto.
```

### Conceptos principales

```Bash
- Sprint

  Periodo de 1 a 4 semanas donde se desarrolla una parte del producto.

- Product Backlog

  Lista completa de tareas o requisitos.

- Sprint Backlog

  Tareas seleccionadas para un sprint.

- Sprint Review
  Revisión final del trabajo realizado.

- Daily Scrum
  Reunión diaria breve para ver avances y problemas.
```

### ¿Que es Kanban?

```Bash
Kanban es un método de gestión de tareas basado en la visualización del trabajo mediante un tablero.
```

### Funcionamiento

```Bash
El trabajo se organiza en columnas como:

- Pendiente
- En proceso
- Finalizado
```

Las tareas se representan como tarjetas que se mueven entre columnas.

### Objetivo

```Bash
 Mejorar la organización y mantener un flujo continuo de trabajo.
```

### Caracteristicas

```Bash
- No utiliza sprints
- Flujo continuo de tareas
- Limitación de tareas en proceso (WIP)
```

---

## Diferencias entre Scrum y Kanban

```Bash
Scrum se enfoca mas en la estructura y adaptacion predefinida, mientras que Kanban
deja mas libertad (Flexibilidad) tal como: BD estructurada (Scrum) y no estructurada (Kanban)
```

---

### ¿Cuando usar cada metodología?

### Scrum

Se usa cuando:

```Bash
- El proyecto necesita estructura
- Se trabajan funcionalidades grandes
- Hay un equipo definido
- Se requieren entregas periódicas
```

**Ejemplo:** desarrollo de una aplicación.

### Kanban

Se usa cuando:

```Bash
- El trabajo es continuo
- Las prioridades cambian frecuentemente
- Se necesita flexibilidad
```

**Ejemplo:** mantenimiento o soporte técnico.

---

## Resumen

```Bash
- Agile: enfoque flexible de desarrollo

- Scrum: trabajo por ciclos (sprints)

- Kanban: flujo continuo visual

- Scrum vs Kanban: estructura vs flexibilidad
```

- Uso:

```Bash
  - Scrum → proyectos organizados
  - Kanban → tareas continuas
```
